Deploy Middleman to Github User Pages

27 Mar 2014

There are required structural difference between Github Project & User Pages. Deploying Middleman to Github Project Pages is simple with the gem middleman-gh-pages. But I'm here to explain how to deploy middleman to Github User Pages.

A quick overview of Github User Pages

  • the url is simply: http://username.github.io.
  • You can only have one of these per github account
  • Generated site must be pushed to 'master' branch (LAME!).
  • Deploying requires this blog post

Setting up Git

Remember Github User Pages require the generated site to be in branch 'master'. So we'll create our middleman repo in branch 'source'.

Clone Middleman or middleman-loaded.. my version of it. :)

git clone git@github.com:middleman/middleman.git username.github.io

or 

git clone git@github.com:dnajd/middleman-loaded.git username.github.io

Remove the .git folder

cd username.github.io
rm -rf .git

Init a new git repo with branch 'source'

git init .
git checkout -b source
git add .
git commit -m"forked middleman for github user pages"

Now browse to Github and manually create your repository. It must be named username.github.io.git, where the username is your github username.

Add a remote & push 'origin source'

git remote add origin git@github.com:username/username.github.io.git
git push -u origin source

Create 'origin master', push and the remove it locally (trust me).

git branch master
git push origin master
git branch -d

Middleman Configuration

You'll want to add the gem middleman-deploy to your app.

And the following in config.rb:

# github deploy
activate :deploy do |deploy|
  deploy.method = :git
  deploy.build_before = true
  deploy.branch   = "master"
  deploy.remote   = "git@github.com:username/username.github.io.git"
end

And Deploy

Remember to build gems 

bundle install

Each time you make changes you'll do the usual commit / push; and then run this to publish to Github User Pages

bundle exec middleman deploy
